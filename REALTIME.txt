
OAUTH2

"In my logistics platform, I implemented OAuth2 Resource Server with JWT by creating a /api/auth/login endpoint that validates credentials and returns a JWT token. Drivers store that token and send it in every API call (like updating POD). Spring Security validates the JWT signature on each request without storing session state, and I used @PreAuthorize to restrict endpoints by role (only drivers can update POD)."


OAuth2 is a protocol (framework for delegated authorization), while JWT is a token format (compact, signed way to carry user data).

"Yes, OAuth2 supports role-based access control. In my logistics platform, the JWT token includes a role claim (DRIVER/ADMIN). When a driver calls /api/orders/{id}/pod, Spring Security extracts the role from JWT using JwtAuthenticationConverter, then checks @PreAuthorize("hasRole('DRIVER')") to grant or deny access."

Spring Cloud Resilience4j - All Concepts
1. Circuit Breaker рҹ”Ң
Most important pattern - stops calling failing service, returns fallback

Aspect	Details
What it does	When service fails вҶ’ circuit opens вҶ’ stop sending requests 
States	CLOSED (normal) вҶ’ OPEN (failing) вҶ’ HALF_OPEN (testing) 
Why use	Prevent cascading failures in microservices 
Real-Time Usage (Your Project):

java
// Configuration
@Configuration
public class ResilienceConfig {

    @Bean
    public CircuitBreakerFactory circuitBreakerFactory() {
        return new Resilience4JCircuitBreakerFactory();
    }
}

// Service with Circuit Breaker
@Service
public class DriverNotificationService {

    @Autowired
    private PushNotificationService pushService;

    @CircuitBreaker(name = "push-notification", fallbackMethod = "fallbackNotification")
    public void sendToDriver(String driverId, String message) {
        // Call push notification service (may fail)
        pushService.sendPush(driverId, message);
    }

    // Fallback method when circuit opens
    public void fallbackNotification(String driverId, String message, Exception ex) {
        // Use SMS instead when push fails
        smsService.sendToDriver(driverId, message);
    }
}

2. Retry рҹ”„
Automatically retry failed request - useful for transient errors

Aspect	Details
What it does	Failed request вҶ’ retry N times with delay 
Why use	Transient errors (network timeout) вҶ’ retry succeeds 
Config	Max retries, retry interval, exception filter 
Real-Time Usage (Your Project):

java
// Configuration
@Bean
public RetryRegistry retryRegistry() {
    RetryConfig config = RetryConfig.custom()
        .maxAttempts(3)
        .waitDuration(Duration.ofSeconds(2))
        .retryOn(exception -> exception instanceof HttpRequestTimeoutException)
        .build();
    return RetryRegistry.of(config);
}

// Service with Retry
@Service
public class OrderStatusService {

    @Autowired
    private TrackingService trackingService;

    @Retry(name = "tracking-service")
    public void updateOrderStatus(String orderId, String status) {
        // Call tracking service (may timeout)
        trackingService.updateStatus(orderId, status);
    }
}

3. Rate Limiter рҹҡҰ
Limit request rate - prevent system overload

Aspect	Details
What it does	Limit requests per second (token bucket algorithm) 
Why use	Prevent system overload during peak traffic 
Config	Requests per second, timeout, limit refresh period 
Real-Time Usage (Your Project):

java
// Configuration
@Bean
public RateLimiterRegistry rateLimiterRegistry() {
    RateLimiterConfig config = RateLimiterConfig.custom()
        .limitRefreshPeriod(Duration.ofSeconds(1))
        .limitForPeriod(100) // 100 requests per second
        .timeoutDuration(Duration.ofMillis(500))
        .build();
    return RateLimiterRegistry.of(config);
}

// Service with Rate Limiter
@Service
public class DriverAssignmentService {

    @Autowired
    private BiddingService biddingService;

    public void assignOrderToDriver(String orderId, String driverId) {
        // Call bidding service (rate limited)
        biddingService.submitBid(orderId, driverId);
    }
}

4. Timeout вҸұпёҸ
Set maximum time for request - avoid waiting forever

Aspect	Details
What it does	Request fails if takes longer than timeout 
Why use	Slow services вҶ’ don't block entire system 
Config	Timeout duration, cancel running on timeout 
Real-Time Usage (Your Project):

java
// Configuration
@Bean
public TimeoutRegistry timeoutRegistry() {
    TimeoutConfig config = TimeoutConfig.custom()
        .timeoutDuration(Duration.ofSeconds(5)) // 5 sec timeout
        .cancelRunningOnTimeout(true)
        .build();
    return TimeoutRegistry.of(config);
}

// Service with Timeout
@Service
public class RouteMasterService {

    @Autowired
    private RouteService routeService;

    @Timeout(name = "route-service")
    public Route getRouteMaster(String routeId) {
        // Call route service (may be slow)
        return routeService.getRoute(routeId);
    }
}

5. Bulkhead (Isolation) рҹҡ§
Isolate service calls - prevent one failure affecting others

Aspect	Details
What it does	Limit concurrent calls per service (thread pool/semaphore) 
Why use	Prevent cascade: one service failure вҶ’ system crash 
Config	Max concurrent calls, max wait duration 
Real-Time Usage (Your Project):

java
// Configuration
@Bean
public BulkheadRegistry bulkheadRegistry() {
    BulkheadConfig config = BulkheadConfig.custom()
        .maxConcurrentCalls(100) // 100 concurrent calls
        .maxWaitDuration(Duration.ofMillis(500))
        .build();
    return BulkheadRegistry.of(config);
}

// Service with Bulkhead
@Service
public class OrderAnalyticsService {

    @Autowired
    private AnalyticsService analyticsService;

    @Bulkhead(name = "analytics-service")
    public void aggregateOrderStats(String date) {
        // Call analytics service (bulkhead protected)
        analyticsService.aggregateStats(date);
    }
}
