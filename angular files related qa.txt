What is the purpose of angular.json?
ngular.json is the workspace configuration file. It defines:
    
• Build targets (build, test, serve, lint) with options and file replacements
• Asset paths and budget limits
• Output paths for compiled files
• Environment-specific configurations (production, development)


What does main.ts do in Angular 17?

main.ts is the entry point of the application. In Angular 17 with standalone components it uses bootstrapApplication() instead of the old platformBrowserDynamic().bootstrapModule().



What is app.config.ts and why was it introduced?

Introduced in Angular v15+ and made the default in v17, app.config.ts holds the root application configuration for standalone apps.


What is the role of tsconfig.json vs tsconfig.app.json vs tsconfig.spec.json?
Angular 17 projects contain three TypeScript configs:

tsconfig.json — base config shared by everything; sets strict, target, path aliases.

tsconfig.app.json — extends base; used for building the app; excludes test files.

tsconfig.spec.json — extends base; used by Karma/Jest; includes *.spec.ts files.


How does app.routes.ts differ from the old RouterModule approach?

In Angular 17 standalone apps, routes are plain Routes arrays — no RouterModule.forRoot() needed in a module.


What are the environment files and how do you use them in Angular 17?
Environment files live in src/environments/ and hold build-time constants:
In Angular 17, environments are not generated by default — run ng generate environments to create them. They are swapped via fileReplacements in angular.json at build time.

How do you organise server-side rendering (SSR) files added by Angular 17?

app.config.server.ts merges the browser config with server-only providers (e.g., provideServerRendering()). The build outputs two bundles: browser/ and server/ under dist/.


What is the significance of the .angular/ cache folder introduced in recent Angular versions?

angular/cache/ is a persistent build cache folder created by Angular CLI (v13+, used heavily in v17). It stores:

• Compiled TypeScript output fragments
• Webpack/esbuild module graph snapshots
• Babel transformation results



What is the difference between styles.css (global) and component-level styles?
Component-level styles (e.g., app.component.css) are encapsulated using Angular's ViewEncapsulation, which adds unique attribute selectors to prevent leaking.












How does the esbuild-based builder change the output structure in Angular 17?

Angular 17 switched the default builder from Webpack to esbuild (@angular-devkit/build-angular:application). Key differences in output:
esbuild produces named chunk files (e.g., chunk-ABCD1234.js) instead of Webpack's numbered chunks. Builds are 2–4× faster and output is better tree-shaken. The outputMode: "static" option also enables pre-rendering into the browser/ folder.
