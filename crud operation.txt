**Optional (but highly recommended for real-world applications): Data Transfer Objects (DTOs)**

For a more robust API, you would often introduce DTOs to separate the internal entity model from the external API representation.

  * **`ProductRequestDTO`:** For incoming data (e.g., when creating or updating a product). It might have fewer fields than the `Product` entity (e.g., no `id` for creation).
  * **`ProductResponseDTO`:** For outgoing data (e.g., when returning product details). It might flatten relationships or include calculated fields.

You would then convert between DTOs and Entities in the Service Layer. This adds a bit more code but provides better control over your API contract and prevents exposing internal database details.

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;
import java.util.List;
import java.util.Optional;
import java.util.stream.Collectors;

@Service
public class ProductService {

    private final ProductRepository productRepository;

    @Autowired
    public ProductService(ProductRepository productRepository) {
        this.productRepository = productRepository;
    }

    // --- Mappers ---
    private ProductDTO mapToDTO(Product product) {
        return new ProductDTO(product.getId(), product.getName(), product.getPrice(), product.getDescription());
    }
    
    private Product mapToEntity(ProductDTO dto) {
        return new Product(dto.getName(), dto.getPrice(), dto.getDescription());
    }

    // --- Custom Derived Methods Implementation ---

    @Transactional(readOnly = true)
    public Optional<ProductDTO> getProductByName(String name) {
        return productRepository.findByName(name).map(this::mapToDTO); // Uses findByName
    }

    @Transactional(readOnly = true)
    public List<ProductDTO> getProductsMoreExpensiveThan(Double price) {
        return productRepository.findByPriceGreaterThan(price).stream() // Uses findByPriceGreaterThan
                .map(this::mapToDTO)
                .collect(Collectors.toList());
    }

    @Transactional(readOnly = true)
    public List<ProductDTO> searchProductsByKeyword(String keyword) {
        return productRepository.findByNameContainingIgnoreCase(keyword).stream() // Uses findByNameContainingIgnoreCase
                .map(this::mapToDTO)
                .collect(Collectors.toList());
    }
    
    // --- Example CRUD Method ---
    
    @Transactional
    public ProductDTO createProduct(ProductDTO productDTO) {
        if (productDTO.getPrice() == null || productDTO.getPrice() < 0) {
             throw new IllegalArgumentException("Product price must be non-negative.");
        }
        Product newProduct = mapToEntity(productDTO);
        Product savedProduct = productRepository.save(newProduct);
        return mapToDTO(savedProduct);
    }
}

import org.springframework.stereotype.Service;

/**
 * Service A's business logic layer. 
 * This class encapsulates the synchronous call to Service B using RestTemplate.
 */
@Service
public class UserService {

    // Target URL of the external service (assuming Service B runs on port 8081)
    private static final String EXTERNAL_SERVICE_BASE_URL = "http://localhost:8081/external/users";
    
    // In a production environment, RestTemplate would be configured as a Bean and injected.
    private final RestTemplate restTemplate = new RestTemplate();

    /**
     * Calls Service B to retrieve user details by ID.
     * @param id The ID of the user to fetch.
     * @return The UserDTO retrieved from the external service.
     * @throws RuntimeException if communication fails.
     */
    public UserDTO fetchUserDetails(Long id) {
        
        String targetUrl = EXTERNAL_SERVICE_BASE_URL + "/" + id;
        System.out.println("UserService (RestTemplate): Calling external service at: " + targetUrl);

        try {
            // Make the synchronous service-to-service call
            UserDTO externalUserDetails = restTemplate.getForObject(
                targetUrl, 
                UserDTO.class // The expected return type
            );
            
            if (externalUserDetails == null) {
                // If the response body is empty, handle it as a specific error
                throw new RuntimeException("External service returned an empty response body.");
            }
            
            return externalUserDetails;

        } catch (HttpClientErrorException e) {
            // Handles 4xx client errors (e.g., 404 Not Found) returned by Service B
            System.err.println("External Service returned HTTP error: " + e.getStatusCode());
            throw new RuntimeException("External Service Error: " + e.getMessage());

        } catch (ResourceAccessException e) {
            // Handles connection problems (e.g., Service B is down or port is wrong)
            System.err.println("Connection Error: External Service B is unavailable.");
            throw new RuntimeException("Service Unavailable: Could not connect to external user service.");
            
        } catch (Exception e) {
            // General exception handler
            System.err.println("Unexpected error during external service communication: " + e.getMessage());
            throw new RuntimeException("Communication Failure.");
        }
    }
}

### **REST API Endpoint for Creating a New Order**

**Scenario:** We want to allow a client to create a new order by providing the customer's ID and a list of product IDs included in the order. The total amount and order date will be handled by the backend.

#### **1. Request DTO (Data Transfer Object)**

This DTO will represent the structure of the JSON (or XML) payload that the client sends in the request body.

```java
// OrderRequestDTO.java
import jakarta.validation.constraints.NotNull;
import jakarta.validation.constraints.NotEmpty;
import jakarta.validation.constraints.Positive;
import java.util.List;

public class OrderRequestDTO {

    @NotNull(message = "Customer ID cannot be null")
    @Positive(message = "Customer ID must be a positive number")
    private Long customerId;

    @NotEmpty(message = "Product IDs list cannot be empty")
    private List<Long> productIds;

    // --- Constructors ---
    public OrderRequestDTO() {}

    public OrderRequestDTO(Long customerId, List<Long> productIds) {
        this.customerId = customerId;
        this.productIds = productIds;
    }

    // --- Getters and Setters (Essential for Spring's deserialization) ---
    public Long getCustomerId() {
        return customerId;
    }

    public void setCustomerId(Long customerId) {
        this.customerId = customerId;
    }

    public List<Long> getProductIds() {
        return productIds;
    }

    public void setProductIds(List<Long> productIds) {
        this.productIds = productIds;
    }

    // --- toString() (Optional, but good for logging/debugging) ---
    @Override
    public String toString() {
        return "OrderRequestDTO{" +
               "customerId=" + customerId +
               ", productIds=" + productIds +
               '}';
    }
}
```

**Explanation of DTO and Annotations:**

  * **`OrderRequestDTO`**: This class serves as the blueprint for the incoming JSON payload. It's a plain Java object.
  * **`@NotNull`**: Ensures that the `customerId` field itself is not `null`.
  * **`@Positive`**: Ensures that the `customerId` is a positive number.
  * **`@NotEmpty`**: Ensures that the `productIds` list is not `null` and not empty.
  * **`List<Long> productIds`**: Represents the collection of product identifiers.
  * **Getters and Setters**: Crucial for Spring's ability to automatically deserialize the JSON request body into an `OrderRequestDTO` object using a library like Jackson.

import java.util.List;

@Repository
public interface ProductRepository extends JpaRepository<Product, Long> {

    /**
     * Finds all products that are cheaper than a given price
     * AND belong to a specific category.
     *
     * Spring Data JPA automatically generates the query based on the method name.
     * 'LessThan' for price < value
     * 'And' to combine conditions
     * 'ByCategory' for category = value

     * @param price The maximum price (exclusive).
     * @param category The category name.
     * @return A list of matching Product entities.

    List<Product> findByPriceLessThanAndCategory(BigDecimal price, String category);
