SQL Interview Questions and Answers
Question 1: Retrieve all employees along with their department names.

Concept: INNER JOIN

Query:

SQL

SELECT
    e.first_name,
    e.last_name,
    e.salary,
    d.department_name
FROM
    Employees AS e
INNER JOIN
    Departments AS d ON e.department_id = d.department_id;
Explanation: An INNER JOIN is used to combine rows from Employees and Departments tables based on a matching department_id in both tables. Only employees who are assigned to a department will appear in the result.

Question 2: Find all departments that currently have no employees.

Concept: LEFT JOIN and WHERE IS NULL

Query:

SQL

SELECT
    d.department_name
FROM
    Departments AS d
LEFT JOIN
    Employees AS e ON d.department_id = e.department_id
WHERE
    e.employee_id IS NULL;
Explanation: A LEFT JOIN includes all rows from the Departments table. If a department has no matching employees, the employee_id (and other Employees table columns) will be NULL. We then filter for these NULL values to find departments without employees.

Question 3: Calculate the average salary for each department. Include departments even if they have no employees (average salary would be NULL or 0).

Concept: GROUP BY, Aggregate Functions (AVG), LEFT JOIN

Query:

SQL

SELECT
    d.department_name,
    COALESCE(AVG(e.salary), 0) AS average_salary -- COALESCE handles NULL for departments with no employees
FROM
    Departments AS d
LEFT JOIN
    Employees AS e ON d.department_id = e.department_id
GROUP BY
    d.department_name
ORDER BY
    d.department_name;
Explanation: We use a LEFT JOIN to ensure all departments are included. AVG(e.salary) calculates the average salary for each group (department). COALESCE(AVG(e.salary), 0) replaces NULL average salaries (for departments with no employees) with 0 for cleaner output. GROUP BY department_name aggregates the salaries for each distinct department.

Question 4: Find departments where the average salary is greater than $70,000.

Concept: GROUP BY, HAVING, Aggregate Functions (AVG)

Query:

SQL

SELECT
    d.department_name,
    AVG(e.salary) AS average_salary
FROM
    Departments AS d
INNER JOIN
    Employees AS e ON d.department_id = e.department_id
GROUP BY
    d.department_name
HAVING
    AVG(e.salary) > 70000;
Explanation: First, we INNER JOIN to get employees and their departments. Then, we GROUP BY department to calculate the AVG(e.salary) for each. Finally, HAVING filters the groups based on the condition that their average salary is greater than $70,000. HAVING is used for filtering on aggregated results, whereas WHERE is for filtering individual rows before aggregation.

Question 5: Find the second highest salary among all employees.

Concept: Subquery with DISTINCT and ORDER BY/LIMIT/OFFSET OR Window Function (DENSE_RANK)

Query (using DENSE_RANK - more robust for ties):

SQL

SELECT salary
FROM (
    SELECT
        salary,
        DENSE_RANK() OVER (ORDER BY salary DESC) as rnk
    FROM
        Employees
) AS ranked_salaries
WHERE rnk = 2;
Explanation:

The inner query uses DENSE_RANK() OVER (ORDER BY salary DESC) to assign a rank to each unique salary in descending order. DENSE_RANK() assigns the same rank to employees with the same salary.

The outer query then selects the salary where the calculated rank (rnk) is 2.

Alternative (using LIMIT/OFFSET - less robust for ties):

SQL

SELECT DISTINCT salary
FROM Employees
ORDER BY salary DESC
LIMIT 1 OFFSET 1;


SELECT DISTINCT salary
FROM Employees
ORDER BY salary DESC
LIMIT 1 OFFSET 2;







Explanation (Alternative): This query first selects distinct salaries, orders them in descending order, and then uses LIMIT 1 OFFSET 1 to get the single salary after skipping the highest one. The drawback is if there are multiple employees with the same highest salary, this will still give you the next distinct salary, not necessarily the second highest value if there are ties at the top.

Question 6: For each customer, list their total number of orders and the total amount spent.

Concept: INNER JOIN, GROUP BY, Aggregate Functions (COUNT, SUM)

Query:

SQL

SELECT
    c.first_name,
    c.last_name,
    COUNT(o.order_id) AS total_orders,
    SUM(o.total_amount) AS total_amount_spent
FROM
    Customers AS c
INNER JOIN
    Orders AS o ON c.customer_id = o.customer_id
GROUP BY
    c.customer_id, c.first_name, c.last_name -- Group by all non-aggregated columns from SELECT
ORDER BY
    total_amount_spent DESC;
Explanation: We join Customers and Orders on customer_id. COUNT(o.order_id) counts the number of orders for each customer, and SUM(o.total_amount) calculates the total money spent. The GROUP BY clause aggregates these results for each unique customer.

Question 7: Add a new employee to the Employees table.

Concept: INSERT

Query:

SQL

INSERT INTO Employees (employee_id, first_name, last_name, department_id, salary, hire_date)
VALUES (10, 'Sophia', 'Chen', 101, 78000.00, '2024-07-25');
Explanation: The INSERT INTO statement specifies the table and the columns you want to populate. The VALUES clause provides the data for those columns in the same order.

Question 8: Update the salary of 'Alice Smith' to $82,000.

Concept: UPDATE, WHERE

Query:

SQL

UPDATE Employees
SET salary = 82000.00
WHERE first_name = 'Alice' AND last_name = 'Smith';
Explanation: The UPDATE statement specifies the table to modify. SET defines the new value for the salary column. The WHERE clause is critical; it ensures that only the row(s) matching the condition (first_name = 'Alice' AND last_name = 'Smith') are updated. Without WHERE, all salaries in the table would be updated!

Question 9: Delete the employee named 'Ivy Garcia'.

Concept: DELETE, WHERE

Query:

SQL

DELETE FROM Employees
WHERE first_name = 'Ivy' AND last_name = 'Garcia';
Explanation: The DELETE FROM statement specifies the table. The WHERE clause identifies the specific row(s) to be removed. Again, be very cautious with DELETE statements, as omitting WHERE will delete all records from the table.

Question 10: Find customers who have placed more than 2 orders.

Concept: INNER JOIN, GROUP BY, COUNT(), HAVING

Query:

SQL

SELECT
    c.first_name,
    c.last_name,
    COUNT(o.order_id) AS number_of_orders
FROM
    Customers AS c
INNER JOIN
    Orders AS o ON c.customer_id = o.customer_id
GROUP BY
    c.customer_id, c.first_name, c.last_name
HAVING
    COUNT(o.order_id) > 2;
Explanation: This query joins customers and orders, groups them by customer, counts their orders, and then filters these groups to show only those with more than 2 orders using the HAVING clause.
